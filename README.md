# DC Maestro

DC Maestro — локально запускаемое SPA-приложение-кокпит для сети ЦОД. Прототип v0.5 делает упор на визуальный слой: глубокий navy-gradient, стеклянные панели, glow-pill статусы и воздушные таблицы при сохранении всей бизнес-логики.

## v0.6 — UX & Navigation
- Перестроена иерархия дашборда: контекстные заголовки, OSI в хедере, «Краткий отчёт» над обновлённой главной панелью с буллетами по SLA, проблемным площадкам и просроченному ТО.
- Навигация следует цепочке «сеть и железо → люди → финансы»: новый порядок пунктов и бейджи-счётчики для площадок, ТО и безопасности.
- Единый заголовок страниц через SectionHeader/PageHeader, повысили контраст подзаголовков и таблиц.
- Строки таблиц теперь кликабельны там, где есть детализация (площадки, предупреждения), с подсветкой и стопом клика на внутренних ссылках.

## Возможности v0.5 — Visual Era
- Новая цветовая глубина (Dark SaaS 2026): navy/ink фон с вертикальным градиентом, неоновые акценты #3EECE2.
- Стеклянные карточки и glow-pill бейджи: градиентные бордеры, мягкие тени, лёгкие hover-свечения.
- Таблицы стали воздушнее: увеличенные заголовки с letter-spacing, подсветка строк при ховере, статус-пиллы с внутренним сиянием.
- Header/Sidebar с подсвеченным активным пунктом, градиентными иконками и плавным смещением на hover.
- Hero-блок на дашборде с насыщенным градиентом, обновлёнными графиками и яркими баррами/линиями.

## Demo для Вис Энергия
- Откройте пункт меню **«Вакансия / Demo»**: там зоны ответственности роли и быстрые переходы по модулям.
- Стартовый маршрут для собеседования: OSI на дашборде → блок «Быстрый переход по ролям».
- Провалитесь в **Площадки ЦОД** и выберите нагруженную площадку (НН-1) для показа инцидентов и KPI.
- Из таблицы предупреждений переходите в **Maintenance** или **Safety**, чтобы разобрать просрочки и инциденты.
- Завершите обзором складов (**Inventory**) и расходов (**Finance**), затем покажите смены в **Personnel**.

## Стек
- Vite + React + TypeScript
- Tailwind CSS для тёмной темы и базовых UI-компонентов
- Recharts для графиков
- Vitest для unit-тестов KPI

## Структура
- `src/app/App.tsx` — каркас приложения с сайдбаром, хедером и роутингом
- `src/modules/*` — экраны Dashboard, Sites (+ SitesDetailPage), Personnel, Maintenance, Inventory, Finance, Safety
- `src/shared/models` — доменные типы
- `src/shared/data` — мок-данные по площадкам, персоналу, инцидентам и т.д.
- `src/shared/lib/kpi.ts` — расчёт KPI и OSI
- `src/shared/lib/strings.ts` — текстовые константы на русском
- `src/shared/hooks/useTableSortAndFilter.ts` — сортировка и поиск по таблицам
- `src/shared/components` — общие элементы UI (Card, Table, KpiBadge, InfoTooltip и др.)

### Мета и заголовки
- Заголовок вкладки браузера автоматически подставляет название активного модуля, а meta description синхронизирован с подзаголовком страницы.

## Запуск
```bash
npm install
npm run dev    # запуск dev-сервера Vite
npm run build  # production-сборка
npm run preview
npm run test   # unit-тесты KPI
```
