# DC Maestro

**DC Maestro** — умный кокпит для руководителя эксплуатации сети ЦОД.  
Локально запускаемое SPA-приложение, которое собирает в одном интерфейсе:

- мониторинг площадок и OSI,
- техническое обслуживание и активы,
- склад и закупки,
- персонал и смены,
- финансы,
- безопасность.

Проект экспериментальный: это демо-панель уровня крупного оператора ЦОД (Ростелеком, МегаФон, IXC и др.), без привязки к реальным данным. Используется как:

- концепт «центрального пульта» для сети дата-центров;
- тестовое/портфолио по UX, архитектуре и doman-driven UI;
- полигон для будущего smart-assistant’а.

## Основные возможности

### Дашборд

- **Operational Strain Index (OSI)** в хедере — сводный показатель нагрузки сети.
- Блок «Обзор состояния сети и эксплуатационных рисков» с KPI по SLA, инцидентам, просроченному ТО и событиям безопасности.
- Панель «Состояние сети сейчас» — аптайм, OSI-индекс, количество площадок вне нормы.
- Таблица текущих предупреждений: инциденты, заявки ТО и события безопасности с приоритетами и быстрым переходом в нужный модуль.
- Блок «Проблемные площадки» и графики по аптайму/нагрузке.

### Площадки ЦОД (Sites)

- Список площадок с показателями **Uptime / Reliability / Capacity** и статусами (стабильно / предупреждение / критично).
- Быстрый поиск по названию или региону.
- Быстрые сводки по сети: средний аптайм, средняя надёжность и загрузка.
- Навигационный сценарий: выбор площадки → переход к инцидентам, ТО, персоналу и финансам этой площадки (WIP).

### ТО и активы (Maintenance)

- Реестр критичных активов (UPS, генераторы, CRAC и др.) с типом, критичностью, площадкой и датой следующего ТО.
- Таблица заявок на ТО: ID, тип (PM/Repair/Inspection), статус, приоритет, связанный актив и площадка, дедлайн.
- Визуальный акцент на просроченных заявках.

### Запасы (Inventory)

- Складские позиции: SKU, русское название, категория (spare_part, tool, consumable, safety), площадка, количество с единицами.
- Модуль закупок: заказы с поставщиком, статусом (черновик, отправлено), датой и ссылкой на позиции.
- Поиск по SKU, названию или поставщику.

### Персонал и смены (Personnel)

- Справочник персонала: имя, роль (technician / engineer / shift_lead / manager), грейд, площадки, ставка ₽/ч.
- Фильтрация по ролям.
- Таблица смен по площадке: дата, начало/конец, план / факт, переработка.
- **Ops Load Index** по площадке — расчёт нагрузки команды по плановым и фактическим часам.

### Финансовая сводка (Finance)

- Таблица финансовых записей: дата, тип (opex/capex), категория (energy, maintenance, staff, hardware), площадка, сумма.
- Быстрый фильтр по площадке и типу.
- Круговая диаграмма структуры затрат и агрегированные Total OPEX / Total CAPEX.

### Охрана труда и безопасность (Safety)

- Реестр событий по охране труда / HSE: тренировки, проверки, инциденты, истечение документов.
- Площадки, даты, статусы (запланировано, завершено, просрочено), ответственные.
- Связка с персоналом и потенциальными задачами на ТО (WIP).

### Вакансия / Demo

- Специальный экран с описанием роли и маршрутом демо для собеседований:
  - OSI на дашборде → проблемные площадки;
  - инциденты и просрочки в Maintenance и Safety;
  - склад и закупки в Inventory;
  - расходы и структура затрат в Finance;
  - смены и загрузка персонала в Personnel.

## Текущий статус и UX-версии

### v0.7 — Luxe Hi-Tech UI

- Пересобрана тёмная палитра: глубокий navy/graphite фон и мягкие матовые панели вместо кислотных градиентов.
- Введён единый фирменный акцент (мягкий голубой) и спокойные статус-цвета для «Стабильно / Внимание / Критично».
- Все карточки KPI и блоки дашборда приведены к единому стилю: тёмные стеклянные панели, аккуратные статус-пиллы, минимум цветного шума.
- Таблицы стали чище: без заливки строк яркими цветами, критичность читается через цветные пиллы и тонкую полосу слева.
- Графики в Finance переехали на однотонную hi-tech палитру и перестали выбиваться из общего интерфейса.
- Обновлён sidebar: спокойные серые элементы, аккуратный акцент для активного пункта.

### v0.6 — UX & Navigation

- Перестроена иерархия дашборда: контекстные заголовки, OSI в хедере, краткий отчёт над основной панелью.
- Обновлена навигация: цепочка «сеть и железо → люди → финансы», бейджи-счётчики для площадок, ТО и безопасности.
- Единый заголовок страниц через `SectionHeader` / `PageHeader`, повышен контраст подзаголовков и таблиц.
- Кликом по строке таблицы открывается детализация (там, где она есть), с подсветкой строки и стопом клика на внутренних ссылках.

### v0.5 — Visual Era

- Глубокий navy-gradient по всему layout + ambient-тени у крупных блоков.
- Стеклянные карточки KPI/инцидентов с мягким свечением при наведении.
- Минимум отвлекающих акцентов в статусах и KPI.
- Менее плотные таблицы, подсветка строк при ховере, выразительные заголовки.
- Обновлённые графики на Recharts.

## Стек

- **Vite + React + TypeScript**
- **Tailwind CSS** — тёмная тема и базовые UI-компоненты
- **Recharts** — графики
- **Vitest** — unit-тесты KPI

---

## Структура проекта

- `src/app/App.tsx` — каркас приложения с сайдбаром, хедером и роутингом.
- `src/modules/*` — экраны:
  - `Dashboard`
  - `Sites` (+ `SitesDetailPage`)
  - `Maintenance`
  - `Inventory`
  - `Finance`
  - `Personnel`
  - `Safety`
  - `About` / `Vacancy`
- `src/shared/models` — доменные типы (площадки, инциденты, активы, смены, финансы и т.д.).
- `src/shared/data` — мок-данные.
- `src/shared/lib/kpi.ts` — расчёт KPI и OSI.
- `src/shared/lib/strings.ts` — текстовые константы на русском.
- `src/shared/hooks/useTableSortAndFilter.ts` — сортировка и поиск по таблицам.
- `src/shared/components` — общие UI-элементы (Card, Table, KpiBadge, InfoTooltip и др.).
